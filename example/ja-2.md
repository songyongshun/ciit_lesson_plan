## 教学目标
### 知识目标
- 理解操作系统与 Linux 的基本概念、历史发展脉络与开源生态的基本构成。
- 区分内核（Kernel）、发行版（Distribution）与常见桌面/服务器环境的职责与差异。
- 理解常见安装环境（虚拟机、WSL、裸机）与系统引导、分区和包管理的基本流程。

### 能力目标
- 能在虚拟机或 WSL 环境下完成基础的 Linux 系统安装与初步配置（用户、网络、更新）。
- 能根据教学或项目需求选择合适的发行版并给出理由（稳定性、社区支持、包管理）。
- 能进行基本的故障排查（镜像校验、网络、引导）并记录安装日志以便复现。

### 素质目标
- 培养开源协作与信息共享意识，鼓励参与社区与查阅官方文档。
- 强化动手实践习惯与安全意识（备份、谨慎分区、最小权限原则）。
- 培养系统化思维，从分层结构理解系统运作并将概念与实践结合。

## 内容展开:教学内容
![系统分层示意](images/lesson-01-diagram.png)

（下文内容较长，覆盖概念、安装流程、常见问题与教学活动）
在本节中，我们从宏观到微观系统地理解 Linux，以及动手完成一次基础安装。首先，解释“什么是 Linux”：严格来说，Linux 是内核（kernel），负责管理硬件资源、进程调度、内存与文件系统。与之配套的工具链（如 GNU 工具集）、包管理器与图形环境构成了我们通常所说的“Linux 发行版”。发行版（Distribution）将内核、系统工具、安装器和应用打包，形成可安装的操作系统镜像。常见发行版包括 Ubuntu（桌面友好）、Debian（稳定）、Fedora（技术前沿）、Arch（滚动更新）以及企业级的 RHEL/AlmaLinux 等。选择发行版时应考虑用途（桌面、服务器或嵌入式）、包管理系统、社区支持与长期维护策略。

为了满足你的要求，这一节将把讲解扩展并详尽阐述各部分内容，从内核设计到安装实务、从包管理到安全配置、并配合课堂活动安排与练习。

一、Linux 的核心概念与历史沿革（详述）

Linux 的起源可追溯到 1991 年 Linus Torvalds 发布的内核原型，随后与 GNU 项目的工具链结合，形成一个完整的类 Unix 操作系统。Linux 内核采用模块化设计，支持可加载模块（modules），使得驱动与功能可以动态加载或卸载。内核负责提供系统调用接口（system call），用户空间的程序通过系统调用请求内核服务，例如文件 I/O、进程创建（fork/exec）、内存分配（mmap/brk）与网络通信。理解内核与用户空间的分离有助于把握为什么某些操作（例如直接访问硬件）必须通过内核完成，而普通应用程序仅需使用标准库（如 glibc）提供的接口。

二、发行版与软件生态

发行版在内核之上集成了若干组件：包管理器（如 APT、DNF、pacman）、初始化系统（systemd、SysV）、图形栈（X11/Wayland）、桌面环境（GNOME、KDE）与常用工具（编辑器、编译器）。不同发行版在包版本、发布策略与支持周期上有显著差异：长期支持（LTS）发行版适合教育与企业部署，而滚动更新适合追求最新特性的开发环境。教学中应引导学生根据用途比较和选择，并演示如何查询发行版信息（`lsb_release -a`、`uname -r`）。

三、安装环境选择：虚拟机 vs WSL vs 裸机

（1）虚拟机：优势是隔离、可回滚（快照）、支持完整安装流程（引导、分区、GRUB）。演示步骤包括：创建虚拟机、挂载 ISO、引导进入安装程序、选择磁盘分区方案（自动或手动）、创建用户并配置网络。示范常见分区布局（/、/home、swap）与磁盘大小建议，并提醒在课堂演示时优先使用自动分区以降低风险。

（2）WSL：适用于仅需命令行与开发环境的场景，优势是轻量、与 Windows 文件系统互操作，但不模拟完整引导过程。演示如何在 Windows 中启用 WSL、安装 Ubuntu 发行版并使用 `wsl --install` 快速搭建环境。

（3）裸机安装：用于深入学习引导、分区与硬件兼容性，但对教学时长与风险要求较高，建议仅作为拓展实践。

四、详细安装步骤与注意事项

- 下载镜像与校验：推荐从官方源下载 ISO，并使用 SHA256 校验确保完整性。
- 创建虚拟机并分配资源：建议至少分配 2 CPU、4GB 内存与 20GB 磁盘用于桌面环境；服务器环境可适当减少图形资源。
- 挂载 ISO 并引导：选择语言、键盘、安装目标与网络设置。
- 分区策略：演示自动分区并说明手动分区（EFI/BIOS 的差异、GPT vs MBR）；说明交换分区与 swapfile 的使用场景。
- 用户创建与安全：创建首个管理员用户，解释 sudo 权限与 root 的区别；建议启用自动更新或教学中示范 `sudo apt update && sudo apt upgrade`。

五、安装后的初步配置

- 系统更新与软件安装：安装常用工具（`htop`、`curl`、`vim`、`git`），配置镜像源以提高下载速度（本地镜像/院校镜像）。
- 网络与时区设置：使用 `timedatectl` 配置时区与 NTP 同步。
- 用户环境配置：解释 shell 初始化文件（`.bashrc`、`.profile`）与环境变量设置，演示如何添加常用 PATH。

六、常见问题排查与故障处理

- 镜像校验失败：建议重新下载并检验。
- 引导失败（GRUB）：演示如何进入救援模式与重装 GRUB。
- 网络不通：检查虚拟机网络模式（NAT/桥接）与宿主机防火墙设置。

七、课堂活动与评估设计

- 教师演示：完整安装流程与首次登录配置。
- 学生实操：分组完成虚拟机安装或 WSL 配置并提交安装报告（截图、命令与日志）。
- 小测验：选择题与简答题覆盖发行版选择理由、安装注意事项与版本检查命令。

八、延伸与资源

- 推荐阅读：官方发行版文档、社区 Wiki（Arch Wiki）、Linux Foundation 教程。
- 实践建议：鼓励学生在虚拟环境中反复实验并记录步骤，培养排查与总结能力。

通过以上扩展内容，学生不仅理解了 Linux 的概念与安装流程，还能在实践中掌握关键技能并形成系统化知识结构。

接着，介绍安装环境选择：对于教学和实验，推荐使用虚拟机（VirtualBox 或 VMware）或 Windows 下的 WSL（Windows Subsystem for Linux）。虚拟机的优势是隔离性好，可模拟整机安装过程，便于练习分区、引导配置与快照管理；WSL 更轻量，适合仅需命令行环境和开发工具的同学，但不展示完整引导与内核配置过程。本课堂将以两条路径并行进行：虚拟机安装 Ubuntu Desktop（演示分区与引导）与 WSL Ubuntu 发行（示范快速获取命令行环境）。

安装前的准备工作包括：检查主机硬件（CPU、内存、磁盘空间）、下载合适的 ISO、创建虚拟机并分配资源、准备网络设置和共享文件夹。实际安装步骤：创建虚拟机——挂载 ISO——引导进入安装程序——选择语言与键盘布局——磁盘分区（教学建议使用自动分区以降低风险，同时演示手动分区的基础概念）——创建第一个用户并设置密码——选择软件包与桌面环境——开始安装并重启。安装完成后，演示首次登录、更新系统（`sudo apt update && sudo apt upgrade`）、安装常用工具（`htop`、`curl`、`vim`）、创建快照。

针对 WSL 的教学，指导学生在 Windows 中启用 WSL 功能、安装适配的发行版（通过 Microsoft Store 或手动导入）、配置默认用户与环境、说明 WSL 与传统虚拟机的差异（如没有独立引导加载程序，采用主机内核适配层）。此外，讲解常见安装中会遇到的问题及排查技巧：镜像校验失败（提示重下或校验 MD5/SHA256）、网络配置问题（使用 NAT/桥接切换）、引导加载器（GRUB）问题（在错误分区安装引导）以及虚拟机分辨率与共享剪贴板设置。

教学环节将穿插概念讲解与即时演示，随后学生动手：第一小组在教师指导下完成虚拟机安装并截图；第二小组完成 WSL 环境配置并运行 `uname -a`、`lsb_release -a` 检查版本信息。课堂末尾安排一个小测验：列出三条选择某个发行版的理由、简述安装时的注意事项、写出检查系统版本的两条命令。通过演示、实践与讨论结合，确保同学在理解概念的同时能够完成实际安装，为后续课程（如命令行、文件系统与用户管理）打下坚实基础。
